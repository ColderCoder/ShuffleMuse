# Web UI 用户指南

ShuffleMuse 当前界面使用英文。本指南按实际页面和常见操作说明其行为。

## 打开应用

### 认证检查

浏览器首次打开页面时会请求 `/api/status`，前端认证状态分为：

- `checking`：正在检查服务；
- `authenticated`：已经登录，或服务没有启用密码；
- `anonymous`：服务要求密码但当前 Session 无效；
- `unavailable`：状态接口不可达，此时显示 Retry，而不是错误地当作无密码模式。

密码模式下，访问受保护页面会跳转到 `/login?redirect=...`；登录成功后返回原目标。普通 Session 有效 1 小时，勾选 Remember me 后有效 30 天，服务重启会使两者立即失效。

连续错误密码达到服务器阈值时，登录页读取 `Retry-After`，显示剩余秒数并暂时禁用提交按钮。封禁期间即使输入正确密码也必须等待到期。

### 曲库初始化

HTTP 服务会先开始监听，再在后台完成首次扫描：

- 扫描中显示 “Scanning library”；
- 扫描失败显示错误和 “Retry scan”；
- 首次成功后进入页面并生成随机队列，但不会自动播放；
- 后续状态每 2 秒轮询一次；短暂失败时保留最后成功状态并显示警告。

## 页面导航

| 页面 | 路径 | 用途 |
| --- | --- | --- |
| Home | `/` | 当前歌曲、封面和按标签过滤队列 |
| Browse | `/browse` | 按目录浏览、预览、下载、播放和管理标签 |
| Tags | `/tags` | 查看标签云、标签文件和导出 CSV |
| Graveyard（Tags 子页） | `/tags/graveyard` | 查看并清除缺失文件的孤立标签记录；旧 `/graveyard` 会重定向到此处 |
| Login | `/login` | 密码模式登录 |

桌面端一级导航和搜索位于顶部，播放列表固定在右侧。Graveyard 不再占用一级导航，而是与 Tags 共用 Tags 功能区内的二级页签。窄屏下一级导航、搜索改为展开面板，Tags 与 Graveyard 仍使用同一组二级页签，播放列表移入内容区域。

## 播放与队列

### 初始队列

曲库就绪后，服务端基于当时的不可变曲库快照生成随机队列；浏览器只取得当前 200 首页面，并最多缓存 5 个队列页面，不会先下载整个曲库。队列记录创建时的 generation；重扫不会暗中改写已经存在的顺序，只会把离线项标为不可用。

播放列表界面每次渲染 200 首，提供：

- `Previous 200`；
- `Next 200`；
- `Jump to current`；
- `Randomize`。

`Randomize` 会让服务端重新生成并打乱当前标签范围。若原来正在播放，会立即播放新队列第一首；若原来暂停，只准备新的第一首。

### 选择歌曲

- 点击队列内歌曲：播放该位置；
- 从 Search 或 Browse 播放队列外歌曲：将其插到队首并立即播放；
- `Next` 到达队尾：重新随机生成同一标签范围并播放；
- 队列第一首按 `Previous`：回到本曲开头；其他位置按 `Previous`：播放上一首。

异步队列请求带有播放意图 epoch。用户显式选歌、切换歌曲、退出或重置后，迟到的旧队列不会覆盖当前选择。

### Original 与 Opus

| 模式 | 行为 | 跳转方式 |
| --- | --- | --- |
| Original | 浏览器直接读取原文件 | HTTP Range / 原生 seek |
| Opus | FFmpeg 实时输出 Ogg Opus | 以 `start=<秒>` 重启转码 |

切换模式时保留当前绝对播放位置和播放/暂停状态。Original 不占 FFmpeg 并发；Opus 会占一个共享媒体槽。

播放器的 Opus 模式会显示服务器当前生效的转码码率；管理员修改 `MUSIC_OPUS_BITRATE` 后，已认证状态接口和界面文案会同步更新。

### 浏览器保存的设置

以下设置保存在当前浏览器 `localStorage`：

- 音量，默认 80%；
- 静音状态，默认关闭；
- 串流模式，默认 Original。

随机队列、当前歌曲和播放位置不会作为服务端 Session 持久化。

### 收藏

心形按钮通过普通标签 `favorite` 实现：

- 空心：为当前文件添加 `favorite`；
- 实心：移除 `favorite`；
- Tags 页面和 Home 的标签过滤器会像处理其他标签一样处理它。

## Home

Home 显示：

- 可选标签过滤器；
- 当前曲名和相对路径；
- 同目录 `cover.jpg` / `cover.png`，不存在时回退内嵌封面；两者都不可用时显示缺省封面；
- 当前路径到 Browse 目录的链接。

切换标签过滤器后，队列严格只包含该标签的曲目。当前歌曲已经带有该标签时，
会保留歌曲、播放位置和播放状态；不带该标签时会切换到新队列第一首，播放中
切换会继续播放新曲，暂停状态下则保持暂停。选择 `favorite` 后，播放到队尾会
继续创建仅含 favorite 的新随机队列。

新曲目的封面请求会延迟 500 ms 并以低优先级发出，避免抢在音频前面。同一目录封面成功加载后，切换到该目录的下一首会继续使用相同图片节点和稳定 URL；刷新页面后由浏览器的一小时私有缓存复用。

## Search

搜索只匹配不含扩展名的曲名，大小写不敏感：

- 输入停止 300 ms 后查询；
- 每页 50 条；
- 显示已加载数和总数；
- `Load 50 more` 追加下一页并按文件 ID 去重；
- 新查询发出后会取消旧 HTTP 请求；即使底层响应恰好迟到，也不会回写。

键盘操作：

| 按键 | 动作 |
| --- | --- |
| `ArrowDown` / `ArrowUp` | 移动活动结果 |
| `Enter` | 播放活动结果 |
| `Escape` | 收起结果 |

选择搜索结果后立即播放；移动端搜索面板同时关闭。

## Browse

Browse 使用 `/browse?dir=<相对目录>` 表示当前位置，面包屑可返回任意上级目录。目录和文件合计每页 50 项；翻页会替换当前页，进入新目录时先取消旧请求并清空旧内容，避免继续扫描或显示错误目录。

### 文件操作

| 文件类型 | Preview | Download | Play |
| --- | --- | --- | --- |
| 支持的音频 | 否 | 原文件 | 是 |
| JPG/JPEG/PNG/GIF/WebP/AVIF/BMP | 是 | 原文件 | 否 |
| TXT/CUE/LOG/M3U/M3U8/MD/NFO/LRC/JSON，且不超过 2 MiB | 是 | 原文件 | 否 |
| PDF | 浏览器 iframe | 原文件 | 否 |
| 其他普通文件 | 否 | 原文件 | 否 |

Preview 对话框打开后会：

- 把初始焦点放到关闭按钮；
- 将背景设为 inert；
- 在对话框内循环 Tab / Shift+Tab；
- 允许 Escape 关闭；
- 关闭后恢复原焦点。

Browse 会隐藏点文件和点目录、`*:Zone.Identifier`、`Thumbs.db` 和 `desktop.ini`。它仍可展示和下载音乐根目录中的其他普通文件，因此管理员不应把密钥、数据库备份或私密配置放在音乐目录内。

### 管理标签

音频文件行的 Tags 操作会按需载入该文件的全部标签。Web UI 标签名规则：

- 最长 50 个 ASCII 字符；
- 只能使用字母、数字、`.`、`-`、`_`；
- 不允许空格、斜杠和反斜杠；
- 当前文件上大小写不敏感地阻止重复名称。

添加或移除后，文件行和全局标签云会刷新。

### Rescan

Browse 标题区的重扫按钮只提交后台扫描请求。后续重扫期间：

- 旧曲库快照继续可用；
- 当前歌曲、进度、音量、模式和队列不改变；
- 只有音频路径集合变化时 generation 递增；
- 成功发布后 Browse、Tags 和 Graveyard 根据 generation 刷新。

默认 `MUSIC_RESCAN_INTERVAL=0`，因此冷启动首次扫描完成后只通过此按钮手动重扫；管理员可配置正数周期重新启用定时重扫。Graveyard 不提供第二个重扫入口。

## Tags

Tags 功能区的 Tags 页签显示当前在线文件的标签云和数量。只属于缺失文件的标签不会出现在标签云中，但仍保存在数据库和 CSV 导出中。

点击标签后只取得当前 200 条服务端分页，并使用 `Previous` / `Next` 翻页；浏览器不会预取并保留该标签的全部在线文件。每行可播放歌曲或管理其全部标签。移除当前选择的标签后，该文件立即从当前页移除，标签总数同步减少。

### Export CSV

标题右侧的 `Export CSV` 在标签云和标签详情中都可用。它始终导出整个标签数据库，不按当前标签过滤。

下载文件名为 `shufflemuse-tags.csv`，格式为：

| 列 | 含义 |
| --- | --- |
| `filepath` | 相对音乐根的路径，旧未迁移记录也可能是绝对路径 |
| `name` | 不含扩展名的名称 |
| `dir` | 路径的目录部分；当前记录通常为相对目录，旧绝对记录会保留绝对目录 |
| `tags` | 排序后的标签，以分号连接 |
| `status` | `online` 或 `missing` |

文件包含 UTF-8 BOM，便于电子表格识别非 ASCII 路径。为了阻止公式注入，以 `=`、`+`、`-`、`@`、Tab 或 CR 开头的单元格会增加前导 `'`。

CSV 没有导入功能，也不包含 bbolt 的完整数据库结构，不能替代 `/data/tags.db` 备份。

## Graveyard

Graveyard 是“标签数据库中仍有记录，但当前成功曲库快照没有该路径”的动态列表。常见原因包括：

- 文件被移动、重命名或删除；
- 音乐根目录变化；
- 旧绝对路径尚未成功迁移；
- 当前扫描失败，仍在使用较旧快照。

每项显示名称、目录、完整路径和标签。删除前浏览器会二次确认；删除动作只清除该路径的所有标签关系，不会删除磁盘文件。

若文件在确认与删除之间恢复在线，后端返回 `409 FILE_ONLINE`，页面刷新并保留标签。文件以同一路径恢复并成功重扫后，会自动离开 Graveyard，原标签无需重建。

## 全局快捷键

仅当焦点不在输入框、按钮、链接、选择框、对话框或其他交互控件上时生效：

| 按键 | 动作 |
| --- | --- |
| `Space` | 播放/暂停 |
| `N` | 下一首 |
| `P` | 上一首 |
| `M` | 静音/取消静音 |
| `Escape` | 关闭移动菜单或移动搜索；Preview 中关闭对话框 |

标签云项目支持 Enter 和 Space。所有主要按钮提供可见焦点轮廓和辅助名称。

## Session 失效与退出

除登录失败本身外，任何 API 401 会触发统一 Session 过期流程：

1. 停止音频并清空播放器和队列；
2. 停止状态轮询；
3. 清空标签缓存；
4. 跳转到带原路径 redirect 的 Login。

主动 Logout 也执行同样的本地清理。即使服务器暂时不可达，本地认证状态仍会退出；页面点击处理器目前可能在控制台留下网络 rejection，见[项目审计](PROJECT_AUDIT.md)。
